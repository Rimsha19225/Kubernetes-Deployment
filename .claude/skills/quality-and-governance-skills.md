# QUALITY & GOVERNANCE SKILLS

## SpecComplianceValidationSkill
**Purpose:** Enforce rules and specifications.
**What it does:**
- Checks implementation against approved specs
- Detects missing or incomplete features

**What it does NOT do:**
- Write code
- Define new features

## SpecDriftDetectionSkill
**Purpose:** Prevent silent bugs and scope creep.
**What it does:**
- Detects undocumented behavior
- Identifies scope creep

**What it does NOT do:**
- Implement logic
- Write code

## PhaseReadinessValidationSkill
**Purpose:** Validate phase completion before moving forward.
**What it does:**
- Checks Phase 2 checklist
- Decides if system is "ready to proceed"

**What it does NOT do:**
- Implement features
- Override phase rules
